# 什么是sub-tick（2023）

就需要先知道什么是tick.

客户端（游戏）与服务器通信是需要靠数据包进行的tick就是客户端每秒钟向服务器发送的数据包的数量官匹64tick就是每秒发送64个数据包，也就是15.6毫秒发送1个数据包。这一个数据包包含了上一个tick中玩家的所有操作也就是说，你在游戏中的操作，比如移动 开枪 投掷物等 都需要延迟15.6毫秒才会生效。

而在完美等第三方平台里面，服务器tick数翻倍了，也就是每秒发送128个数据包。

延迟就会降低到7.8毫秒，玩家体验就会变好。但是因为数据包的数量是翻倍的，服务器的压力也会大很多。

因此主流服务器tick数最高也就128。在CS2里面，V社想到了一个新点子，就是sub-tick ，按照他们官方的说法，是什么自适应可变tick。

看似吹的天花乱坠，其实原理很简单，在cs2里面，v设把玩家的操作给“剥离”了。玩家的所有操作不会在同一个数据包中进行打包发送，而是按照重要性分别打包。玩家的移动数据会按照传统的64tick进行上传；玩家的投掷物数据会按照128tick进行上传；而玩家的开火判定会以极高的tick进行上传（按照国外大神的解包，开火信息的数据包每0.22秒就会发送一次，等与4500tick）几乎等于是瞬间判定了，这样做的好处显而易见，开火的判定会变得超级快。

并且因为玩家在一局游戏里面开火的时间其实非常短，未开火的信息其实是空数据包 所以这样做也不会大幅提高服务器负载，但是缺点也显而易见，就是玩家的操作延迟就会有高有低，并不像以前一样均匀一致。

举个很简单的例子，很多玩家觉得cs2里面甩狙甩不动了。

因为在csgo里面，开火是有延迟的，你先按下开火键，此时开火需要下一个tick才会生效，在这个时间间隙里面你可以继续调整准星对准敌人。达到甩狙的效果。

而cs2里面，开火是瞬间生效的。先开火再瞄准的打法就行不通了，必须要瞄准好目标才能开火。

假如你是挨打的一方。因为csgo所有玩家行为是再同一个数据包中处理的，所以敌方大狙开火，你死亡。

这两个事件是同步的。而在cs2里面，敌方大狙开火后，你已经立即被判定死亡了。此时因为玩家移动的数据处理较慢，所以需要等一个tick，死亡的信息才会显示在你的屏幕上。看起来就像是你缩回掩体对面还是把你打死了。

总结一下就是cs2里面，开火的判定更快了，但是因为服务器负载的原因，其他操作的判定还是原来的速度，所以给人的感觉就是不同步。
